# Accessibility and Inclusion

We aim for the calibration and demo experience to be usable by a wide range of operators and participants.

## UI and calibration

- **Large targets** — Calibration corner targets are at least **80×80 px**; “Next” buttons at least **44×44 pt** so the flow is operable with minimal fine motor precision.
- **Few steps** — Calibration is designed to complete in under 2 minutes with minimal typing.
- **Contrast and font size** — We aim for overlay text and UI to meet **WCAG 2.1 Level AA** where applicable: at least **4.5:1** contrast for normal text and a base font size that can be scaled (e.g. ≥ 16 px or relative units). We avoid low-contrast defaults and will verify with contrast checkers as the UI matures.

## Documentation

- Docs use **headings and lists** for structure so screen readers and navigation work well.
- **Images:** When we add screenshots or diagrams, we will provide meaningful alt text or captions.
- **Language** — We use clear, inclusive language and avoid assumptions about ability or background.

## Feedback and limitations

- **Feedback channels** — Issues and discussions are open to all; we enforce the [Code of Conduct](../CODE_OF_CONDUCT.md).
- **Known limitations** — The current interaction model is **single-pointer**: only one pointer is active at a time. A second pointer may be ignored or steal focus. **Workaround:** use one pointer per user or take turns; multi-touch and keyboard shortcuts are on the roadmap.

## References

- We align with general principles of [Inclusive Design](https://guide.inclusivedesign.ca/) where applicable.
- For accessibility standards (e.g. WCAG), we consider contrast and font size in the on-screen UI as the project matures.
