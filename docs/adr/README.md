# Architecture Decision Records (ADR)

This directory holds **Architecture Decision Records** for Smart Pi: short documents that capture important design or scope decisions, context, and consequences.

## Why ADRs

- **Transparency:** Design and scope choices are visible to contributors and users.
- **Context:** Future maintainers can see why a decision was made.
- **Consistency:** New work can align with past decisions or explicitly override them.

## Format

Each ADR is a markdown file (e.g. `0001-use-homography-for-calibration.md`) with:

- **Title** — Short summary of the decision.
- **Status** — Proposed | Accepted | Deprecated | Superseded.
- **Context** — What problem or situation led to this decision.
- **Decision** — What we decided to do.
- **Consequences** — Benefits, drawbacks, and follow-ups.

Optional: **Alternatives considered**, **References**.
